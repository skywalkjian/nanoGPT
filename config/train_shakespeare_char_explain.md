# train_shakespeare_char.py 配置详解

## 概述

`train_shakespeare_char.py` 是 nanoGPT 项目中用于训练**字符级莎士比亚文本生成模型**的配置文件。它定义了一个小型 GPT 模型（"baby GPT"），适合在个人电脑（包括 MacBook）上快速调试和实验。

该文件通过 Python 变量覆盖 `train.py` 中的默认配置。`train.py` 会使用 `exec()` 动态执行配置文件，因此这里定义的变量会直接替换同名的默认值。

### 使用方式

```bash
python train.py config/train_shakespeare_char.py
```

---

## 参数逐行解析

### 1. 输出与日志

```python
out_dir = 'out-shakespeare-char'
```
- **作用**: 模型检查点（checkpoint）和日志的保存目录。
- **默认值**: `'out'`
- **说明**: 使用独立目录避免与其他训练实验的输出混淆。

```python
eval_interval = 250
```
- **作用**: 每隔多少个训练迭代（iteration）进行一次验证集评估。
- **默认值**: `2000`
- **说明**: 因为该小数据集容易过拟合，所以设置较频繁的评估间隔（250 步），以便及时观察模型在验证集上的表现变化。

```python
eval_iters = 200
```
- **作用**: 每次评估时，在验证集上运行多少个 batch 来计算平均 loss。
- **默认值**: `200`（与默认一致）
- **说明**: 200 个 batch 足以得到一个相对稳定的 loss 估计值。

```python
log_interval = 10
```
- **作用**: 每隔多少个迭代在终端打印一次训练 loss。
- **默认值**: `1`
- **说明**: 设为 10 是为了减少终端输出频率，避免刷屏，同时仍能跟踪训练进展。

---

### 2. 检查点保存策略

```python
always_save_checkpoint = False
```
- **作用**: 是否在每次评估后都保存检查点。
- **默认值**: `True`
- **说明**: 设为 `False` 意味着**仅当验证集 loss 有改善时才保存检查点**。由于这个小数据集很容易过拟合，频繁保存没有意义，只保留最优模型即可。

---

### 3. Weights & Biases 日志

```python
wandb_log = False
wandb_project = 'shakespeare-char'
wandb_run_name = 'mini-gpt'
```
- **`wandb_log`**: 是否启用 [Weights & Biases](https://wandb.ai/) 实验追踪。默认关闭，可通过命令行覆盖开启：
  ```bash
  python train.py config/train_shakespeare_char.py --wandb_log=True
  ```
- **`wandb_project`**: W&B 项目名称，用于组织同一实验系列的多次运行。
- **`wandb_run_name`**: 本次运行在 W&B 中的名称标识。

---

### 4. 数据配置

```python
dataset = 'shakespeare_char'
```
- **作用**: 指定使用的数据集名称。对应 `data/shakespeare_char/` 目录，其中包含预处理好的 `train.bin` 和 `val.bin` 文件。
- **说明**: 这是莎士比亚全集的字符级数据集（约 1MB 文本），每个字符被映射为一个 token。

```python
gradient_accumulation_steps = 1
```
- **作用**: 梯度累积步数。实际有效 batch size = `gradient_accumulation_steps * batch_size`。
- **默认值**: `40`（5 * 8）
- **说明**: 设为 1 表示不做梯度累积，每个 micro-batch 直接更新参数。小模型小数据集不需要模拟大 batch。

```python
batch_size = 64
```
- **作用**: 每次前向传播的样本数（micro-batch size）。
- **默认值**: `12`
- **说明**: 字符级模型的 token 很小（单个字符），因此可以使用较大的 batch size。有效 batch size = 1 x 64 = **64 个序列**。

```python
block_size = 256
```
- **作用**: 模型的上下文窗口长度（即序列长度），单位为 token（此处为字符数）。
- **默认值**: `1024`
- **说明**: 模型一次最多看到 256 个前序字符来预测下一个字符。对于字符级模型，256 个字符已经足以捕捉莎士比亚文本的局部模式（单词拼写、短语结构等）。较短的上下文长度也意味着更低的显存需求和更快的训练速度。

---

### 5. 模型架构

```python
n_layer = 6
n_head = 6
n_embd = 384
dropout = 0.2
```

这是一个 **"baby GPT"** 模型，参数规模远小于 GPT-2：

| 参数 | 本配置 | GPT-2 (124M) 默认值 | 说明 |
|------|--------|---------------------|------|
| `n_layer` | 6 | 12 | Transformer 层数 |
| `n_head` | 6 | 12 | 多头注意力的头数 |
| `n_embd` | 384 | 768 | 嵌入维度（隐藏层大小） |
| `dropout` | 0.2 | 0.0 | Dropout 比率 |

**关键细节**:
- **每个注意力头的维度** = `n_embd / n_head` = 384 / 6 = **64 维**（与 GPT-2 一致）。
- **总参数量**: 约 **10M**（相比 GPT-2 的 124M 小了一个数量级）。
- **`dropout = 0.2`**: 预训练大模型通常不用 dropout（设为 0.0），但此处数据集很小（~1MB），模型容易过拟合，因此需要 dropout 进行正则化。

---

### 6. 优化器与学习率调度

```python
learning_rate = 1e-3
```
- **作用**: 最大学习率（峰值学习率）。
- **默认值**: `6e-4`
- **说明**: 小模型可以承受更大的学习率。1e-3 对于这个规模的模型是合理的——更大的学习率意味着更快的收敛。

```python
max_iters = 5000
```
- **作用**: 总训练迭代次数。
- **默认值**: `600000`
- **说明**: 数据集很小，5000 步足以让模型充分学习。更多的迭代只会加剧过拟合。

```python
lr_decay_iters = 5000
```
- **作用**: 学习率余弦衰减的周期长度（以迭代次数计）。
- **默认值**: `600000`
- **说明**: 通常设为与 `max_iters` 相等，确保学习率在训练结束时恰好衰减到 `min_lr`。这遵循了 Chinchilla 论文的建议。

```python
min_lr = 1e-4
```
- **作用**: 余弦衰减的最小学习率（学习率下限）。
- **默认值**: `6e-5`
- **说明**: 经验法则是 `min_lr ≈ learning_rate / 10`，即 1e-3 / 10 = 1e-4。

```python
beta2 = 0.99
```
- **作用**: AdamW 优化器的 beta2 参数（二阶矩估计的指数衰减率）。
- **默认值**: `0.95`
- **说明**: 每次迭代处理的 token 数较少（64 x 256 = 16,384），梯度的方差估计波动较大。较高的 beta2（0.99 vs 0.95）意味着对历史梯度平方的更长记忆，有助于平滑优化过程。

```python
warmup_iters = 100
```
- **作用**: 学习率预热步数。在前 100 步内，学习率从 0 线性增长到 `learning_rate`。
- **默认值**: `2000`
- **说明**: 预热防止训练初期因学习率过大导致的不稳定。对于小模型，100 步的预热已经足够（作者注释也提到"不一定必须"）。

---

### 7. MacBook 额外设置（注释）

```python
# device = 'cpu'  # run on cpu only
# compile = False # do not torch compile the model
```
- 如果在没有 GPU 的 MacBook 上运行，需要取消注释这两行：
  - **`device = 'cpu'`**: 使用 CPU 而非 CUDA。（在 Apple Silicon Mac 上也可以尝试 `device = 'mps'` 来使用 Metal GPU 加速）
  - **`compile = False`**: 禁用 PyTorch 2.0 的 `torch.compile()`。该特性在 CPU 或 MPS 后端上可能不受支持或无法带来加速。

## 学习率调度曲线

训练过程中，学习率按以下策略变化：

```
学习率
  ^
  |         learning_rate = 1e-3
  |        /‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾\
  |       /                      \  (余弦衰减)
  |      /                        \
  |     /                          \
  |    /                            \_____ min_lr = 1e-4
  |   /
  +---+--------+-------------------+-----> 迭代次数
  0  100     ~250               5000
     ↑ warmup   ↑ 开始衰减
```

1. **预热阶段** (0 → 100 步): 学习率从 0 线性增长至 1e-3
2. **余弦衰减阶段** (100 → 5000 步): 学习率从 1e-3 平滑下降至 1e-4

## 总结

这份配置文件体现了几个核心设计思路：

1. **小数据适配**: dropout=0.2 防过拟合，只在验证集提升时保存检查点
2. **快速迭代**: 小模型 + 短上下文 + 少量迭代步数，几分钟内即可在 GPU 上完成训练
3. **适当的超参调整**: 更大的学习率（小模型能承受）、更大的 beta2（小 batch 需要更平滑的优化）
4. **便携性**: 注释中提供了 CPU/MacBook 兼容的配置选项
